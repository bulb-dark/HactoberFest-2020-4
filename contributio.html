Mistakes to avoid while contributing in hactober fest
  1. Avoid submitting low-quality pull requests such as automated changes to fix typos
  2. Do not add your name to a README file or contribute to repositories created solely for Hacktoberfest that offer no real benefit to the open-source community.
  3. Do not submit untested code or contributions that introduce new dependencies without a clear, justifiable reason, as maintainers are wary of potential security risks.
